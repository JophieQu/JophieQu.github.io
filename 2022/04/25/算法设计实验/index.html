测试三

自行设计不同数目的矩阵连乘问题，比较递归和非递归的动态规划矩阵相乘优先级划分（加括号）算法的耗时。并简要分析算法的空间复杂度（存储消耗）。

由于规模较小的话无法体现出递归和非递归做法的差异，所以在测试三（选做）中n的规模会达到1e2及以上。同时因为矩阵数量较多，所以行列信息是随机生成的。

在测试三中，非递归的dp做法和测试一、二中的一致。递归做法由于进行了记忆化处理，所以时间复杂度与非递归做法是同阶的，均为O(n3)O(n^3)O(n3)。但是由于递归调用的栈层较深，以及传值带来的一些时间的消耗，所以随着规模（矩阵个数）的增大，两种做法实际的耗时会有较明显的差异【ps.如果使用四边形不等式（https://oi-wiki.org/dp/opt/quadrangle/#2d1d）来进一步优化矩阵链乘模型，可以将时间复杂度压到O(n2)O(n^2)O(n2)】。实验结果如下表所示：

矩阵数量规模（n） 动态规划矩阵连乘（非递归） 动态规划矩阵连乘（递归）
100 ms 0 ms 1 ms
300 ms 8 ms 31 ms
500 ms 30 ms 134 ms
1000 ms 258 ms 1164 ms
1300 ms 629 ms 2707 ms
1500 ms 1042 ms 6787 ms

至于空间复杂度，显然记忆化递归和非递归都是O(n2)O(n^2)O(n2)；在记忆化递归的时候。因为栈深是常数级别的，所以只需要考虑全局（局部）变量开辟的内存，所以空间复杂度为O(n2)O(n^2)O(n2)。

如果递归做法不做记忆化处理（甚至也不记录路径path），那么递归的栈层会变得很深，甚至栈深上界（即递归树的高）趋近于O(n)O(n)O(n)，可以看到的是，程序机制带来的内存消耗是很大的。那么在考虑栈深的情况下，此时空间复杂度至少为O(n)O(n)O(n)，由于栈的搜索树宽难以估计，所以栈的存储消耗大概介于O(n)O(n)O(n)到O(n2)O(n^2)O(n2)这个区间，空间消耗仍然低于记忆化递归及非递归算法。毕竟非记忆化的递归做法本质上还是一个时间换空间的算法。

#实验题目

动态规划-文本排版

题目描述：

输入n个单词，各单词长度分别为wwi（即每个单词的字母数，i=1…n）。已知页面宽度为wp（即该页每行容许的最大字母数）。要求输出每一行的单词分布。排版单词时，要求不改变各单词原来的顺序，且页面每一行各单词累加的长度不能超过页面宽度。

为了评价每行的排版美观程度，定义如下函数：

badness(i,j)={∞,wlij>wp(wp−wlij)3,elsebadness(i,j)= \begin{cases} \infty,\qquad \qquad \qquad wl_{ij}>wp\\ (wp-wl_{ij})^3,\qquad else \end{cases} badness(i,j)={∞,wlij​>wp(wp−wlij​)3,else​

其中，wlijwl_{ij}wlij​表示第iii个单词到第jjj个单词的累加宽度，wpwpwp为页面宽度。该函数是每一行排版空白数的指标。函数取3次方，是为了增大差异值，凸显一行中存在空白的不美观程度。因此，可以认为badness越小越美观。

例如，页面宽度为20，有5个单词序列[good, good, good, good, good]，各个单词有4个字母，其宽度均为4。则：当一行中只有前2个单词时，badness(1,2) = (20-(4+4))3=1728；当一行中只有前3个单词时badness(1,3) = (20-(4+4+4))3=512；当一行中只有前四个单词时badness(1,4) =(20-(4+4+4+4))3=64。按照badness越小越美观的标准，显然一行中排前四个单词的方案优于前二者。

#实验要求

（1）求解任意输入的nnn个单词的最优排版方案。

（2）假设要求单词之间最少应存在一个空格，则应如何修正求解？给出1~2个解决方案。
